Go笔记（一）——语言概述

发表于 2019-07-15 更新于 2019-07-17 分类于 Golang 阅读次数：

概述

Go 语言是 Google 开源的一门通用型计算机程序设计语言，起源于 2007 年 9 月，并于 2009 年 11 月正式对外发布，其三位创始人都是计算机科学领域的重量级人物：Ken Thompson、Rob Pike 和 Robert Griesemer。作为在近年来快速崛起的编程语言，Go 已经成功跻身主流编程语言的行列，两次荣获 TIOBE 年度语言（分别是 2009 年度和 2016 年度）。

Go 语言是跨平台的，用 Go 编写的程序可以运行在各种类 UNIX 系统（Linux、FreeBSD、OpenBSD 和 Mac OS X）、Plan 9 和 Microsoft Windows 等操作系统上。在 Windows 上编写的 Go 程序，放到 Linux 上也是能够运行的。

在移动端 Go 语言也进行了积极的探索，一开始 Go 语言是不能开发手机移动端应用程序的，后来 Google 建立的 gomobile 项目，使得移动端两大应用平台——Android 和 iOS 上都可以运行其程序。

另外，Go 在 1.11 的版本中引入了 WebAssembly 技术，与 WebAssembly 强强联合，这意味着 Go 语言开始涉足 Web 前端领域，在互联网浏览器也可以运行 Go 编写的程序了。

Go语言的发展过程

Go 语言的推出，旨在不损失应用程序性能的情况下降低代码的复杂性，具有“部署简单、并发性好、语言设计良好、执行性能好”等优势，目前在诸多领域（如云计算、微服务、大数据、区块链、物联网、图像处理、机器学习和人工智能等领域）中我们都能发现它的身影。

与其他高级程序设计语言一样，Go 语言也会从其他成功的语言那里汲取各种优点，取其精华，去其糟粕。下图展示了有哪些早期的编程语言对 Go 语言的设计产生了重要影响。

Go 语言的核心语法与 C 语言的语法表面上看起来很相似，Go 语言有时也会被称为“类 C 语言”或“ 21 世纪的 C”。但这并不是说 Go 语言抄袭了 C语言，而是 Go 语言从 C 语言中继承了很多思想——包括表达式语法、控制流语句、基本数据类型、按值调用的形参传递、指针、代码编译成机器码后的运行效率需要高效、与操作系统可以无缝适配等。

此外，由 Niklaus Wirth 设计的、以 Pascal 为发端的一个语言支流对 Go 语言的设计也产生了重大影响。Modula-2 启发了包概念；Oberon 消除了模块接口文件和模块实现文件之间的差异；Oberon-2 影响了包、导入和声明的语法，并提供了方法声明的语法。

实际上，受到了通信顺序进程（Communicating Sequential Process，CSP）启发一系列名不见经传的研究用语言（如 Squeak、Newsqueak、Alef），同样也对 Go 语言的设计产生了影响，使得 Go 语言相对于当下的程序设计语言显得卓然不群。

除了上述程序设计语言外，Go 语言也从其他语言中借鉴了很多特性，例如，iota 多多少少有点 APL 的影子，而嵌套函数的词法作用域则来自 Scheme（以及由之而来的大部分语言）。

在 Go 语言中，也可以发现全新的变异。Go 语言中新颖的 slice 不仅为动态数组提供了高效的随机访问功能，还允许旧式链表的复杂共享机制。另外，defer 语句也是 Go 语言中新引入的。

Go 语言从其他多种程序设计语言中借鉴了许多优秀的思想，他的种种亮点都受到了广大编程爱好者的追捧。以下是 Go 语言核心特征的一部分：

  • 实现了垃圾回收，即能够自动化的管理内存；
  • 能让你访问底层操作系统，还提供了强大的网络编程和并发编程支持；
  • 简洁、高效、安全；
  • 支持并发编程，
  • 编译迅速，部署便捷，Go 最终生成的是一个二进制可执行文件；
  • 没有类和继承的概念，通过接口（interface）来实现多态性，类型之间也没有层级之说；
  • 实现数据抽象和面向对象的途径是极其灵活的；

书写程序时应遵循的规则

从便于阅读和维护的角度出发，Go 语言要求在书写程序时应遵循以下规则，这些规则是強制的，当不符合规则时编译将会产生错误。

  1. 语句末尾不需要以分号（;）结束。

  2. 大括号（{）不能够换行放置。

  3. if 判断条件和 for 语句不需要以小括号包起来。
